1. HTML 구조
1-1. html 파일은 index.html > template > view 의 부모관계를 가진다.
1-2. index.html에서 ui-router로 각 template를 불러오고 template에서 ng-include를 통해 view를 불러오는 방식을 이용
1-3. /assets/js/angular-app.js 에서 ui-router를 관리한다.
1-4. (1-1 수정) 대부분의 html은 ui-router n단 라우팅에 의해 불러와 진다.

2. layout setting
2-1. $rootScope 변수는 최상위 스코프이며 싱글톤 패턴이 적용된 전역에서 사용가능한 스코프이다.
2-2. router가 변경될 시에 layout.setting 값이 변하고 js/angular-controller.js에 선언되어 있다.
2-3. ***layout 템플릿을 사용한다. js/apps.js에 선언되어 있다.

3. apps.js
3-1. SPMS 구버젼에서 사용했던 템플릿으로 신버젼에서도 그 CSS 이외에 모든 레이아웃의 기초는 이 템플릿을 활용한다.
3-2. js/angular-controller.js 에서 apps.js 의 App 함수의 내부 변수들에 참조하여 layout을 총괄한다.

1. *ui-router
1-1. ui-router에서는 index.html의 <div ng-view> 를 기준으로 화면을 전환하는 라우팅서비스를 제공한다.
1-2. 모든 routing template의 url은 ''로 공백이며 data로 pageTitle 값을 보낸다. pageTitle은 web title을 바꾸는데 이용
1-3. template / templateUrl 을 이용해서 템플릿값을 설정한다.
1-4. ui-router에 app.analysis, app.map, app.elog, app.data, app.mrv 와 같이
template에 <div ng-view></div> 이 선언되어 있는데 이는 다중 라우터를 이용하기 위한 구문으로 보인다.
1-5. state값을 이용하여 라우팅페이지를 구분한다. 이 값은 controller에서 $state 모듈를 로드하여 사용가능하다.

2. *다중 라우팅
2-1. (1-4) 에 언급된 다중 라우팅에 관한 설명이다.
2-2. ParentState.ChildState.GrandChildState 와 같은 형태로 다중 라우팅 구성이 가능하다.
2-3. template에 <div ng-view></div>가 선언된 이유가 다중 라우팅을 구성하기 위한 기반으로 보인다.

3. *resolve (ui-router의 기능)
3-1. resolve는 ui-router를 통해 불러오는 라우팅 템플릿에 넘겨줄 데이터를 로드하는 기능이다.
3-2. resolve를 통해서 해당 template에 해당하는 controller.js 와 assets/plugins 의 필요한 기능을 로드한다.

4. *ocLazyLoad (ui-router library)
4-1. ocLazyLoad는 영역간의 물리적인 번들을 구분할 수 있도록 해준다.
4-2. 이를 통해서 라우팅 될때마다 모든 모듈을 불러오는 것을 막을 수 있고 라우팅 로드 속도를 향상시킬 수 있다.
4-3. 하지만 각각 별도의 번들에 접근 권한을 인증받지 않기 때문에 보안 이슈가 존재한다.

*ocLazyLoad parameters { name - type(defaultValue) }
1. cache - boolean(true) : 'cache: false'는 모든 기본 로더에서 작동하며
이것을 사용하면 로더는 브라우저 캐시를 우회하기 위해 URL에 타임 스탬프를 추가한다.
2. reconfig - boolean(false) : 모듈을 다시 로드하면 기본적으로 예기치 않은 결과가 발생할 수 있으므로
config 블록이 다시 호출되지 않습니다. 이때 config 함수 호출이 반드시 필요하면 'reconfig: true' 속성값을 주면 된다.
3. rerun - boolean(false) : reconfig와 같이 run을 반드시 다시 실행해야할때 'rerun: true'값을 주면 된다.
4. serie(serial) - boolean(false) : 기본적으로 로더는 파일을 병렬로 로드한다.
파일을 직렬로 로드해야하는 경우 'serie : true'를 사용할 수 있습니다.
5. insertBefore - boolean(undifined) : 기본적으로 file은 head 요소의 마지막 요소 앞에 삽입된다.
하지만 'insertBefore: cssSelector'를 사용하여 이를 재정의 할 수 있습니다.

5. $state
5-1. SPMS 페이지는 ui.router 모듈의 state 속성값으로 route를 수행한다.
5-2. state 값은 controller에서 $state 파라미터로 받아올 수 있다. ($state.$current.name)
5-3. angular-controler.js 에서 페이지 변경시 마다 switch문을 통해서 animation과 $rootScope.setting.layout 값 등을 변경한다.
5-4. DOM이 새롭게 로딩될 때마다 documentReady() 메소드를 실행한다.
5-5. documentReady()은 상단 nav bar의 데이터를 관리한다. (update log modal, , now selected ship's departure, arrival point)

$resource
구조 - $resource(url, [paramDefaults], [actions], options);
1. [paramDefaults]
- url parameter의 기본값, 액션 메소드(get, post, delete, put)에 의해 오버라이드 될 수 있다.
- 매개변수가 함수 값일 경우 요청에 대해서 매개변수 값을 필요 시 마다 호출한다.
- 템플릿 /path/:verb와 파라미터 {verb: 'greet', salutation: 'Hello'}의 결과가 URL에는 /path/greet?salutation=Hello 로 들어간다.
- 매개변수 값의 접두사가 '@'면 해당 매개 변수의 값이 데이터 객체의 해당 속성에서 추출된다 무슨말이지?
- 예를들어) {someParam: '@someProp'}이면 someParam의 값은 data.someProp가 된다. 이건 또 무슨말이지?
2. [actions]
- 기본 리소스 작업 집합 외에도 사용할 수 있는 사용자 지정 동작 선언이 있는 해시입니다.
동일한 이름의 기존작업이 잇다면 덮어쓰고 기능을 확장하진 않습니다.
-action : string = action의 이름으로 리소스 객체의 메소드 이름이 된다.
-method : string = 대소문자를 구분하지 않는 HTTP 메소드 (GET, POST, PUT, DELETE, JSON, etc)
-params : object = 이 액션에 대한 사전 바인딩 된 매개 변수의 선택적 세트 매개 변수 값 중 하나가 함수이면 매개 변수 값을 요청할 때마다 호출된다. 이 함수는 현재 데이터 값을 인수로 전달합니다.
-url : string = 특정 URL에 대한 작업을 재정의합니다. url 템플릿은 resource-level과 마찬가지로 지원됩니다.
-isArray : boolean = 참이면 이 액션에 대해 반환 된 객체는 배열입니다.
-transformRequest : {function(data, headersGetter)}

Handsontable library
스프레드 시트를 html Dom에 표시해주는 라이브러리이다.
SPMS 소스에 적용되어 있는 옵션
1. data - data grid(표)에 반영되는 초기 data source. (data grid 편집은 data source을 변경한다.) 배열, 배열 객체, 함수로 선언가능
2. colHeaders - col의 header부분에 표시될 텍스트 boolean 값으로 설정 가능하며 배열로 설정 가능하다. ['shipName', 'shipPrice'] (rowHeaders도 동일하다.)
3. columns - 특정 열에 대한 셀 속성 및 데이터 바인딩을 정의합니다. (이 옵션을 사용하면 고정 된 수의 열을 설정할 수 있다. starCols, minCols, maxCols 옵션은 무시됩니다.)
3-1. data in columns - 데이터의 정렬 순서를 지정한다 {data : index, 'keyName'}
3-2. validation - 셀 검증 과정에서 사용될 함수 또는 정규 표현식 (유효성 검사가 통과되면 true(콜백(true)) 실패시 false(콜백(false))가 리턴된다. 함수의 this는 cellProperties 객체를 가리킨다.)
4. maxRows, maxCols - 행열 최대치 설정값
5. stretchH - 선언된 테이블 너비가 모든 열 너비의 계산 된 합계와 다른 경우 열의 반응 방식을 정의합니다. (none - 짤림, last - 마지막열만 늘림, all - 모든 열을 균등하게 늘림)
6. renderAllRows - true 값을 주게 되면 handsontable의 가상 렌더링 메커니즘이 적용된다. ( 그게 먼지는 모르겟다.)
7. filters - 기본 제공 구성 요소 또는 API를 사용하여 테이블 데이터를 필터링 할 수 있습니다.
8. height - 그리드의 높이, 숫자 혹은 숫자를 리턴하는 콜백을 넣을 수 있다.
9. currnetRowClassName - 'currentRow' 값을 입력하면 해당 row에 클래스를 추가한다.
10. cells - function(row, col, prop) 함수로 정의 가능 지정된 행, 열, 프로퍼티에 대한 셀 속성을 정의합니다.
